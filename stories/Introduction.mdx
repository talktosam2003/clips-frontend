# ClipCash AI Component Library

Welcome to the ClipCash AI component documentation! This Storybook showcases all reusable components used throughout the application.

## 🎨 Design System

ClipCash AI uses a dark-themed design system with:

- **Primary Color (Brand)**: `#00E58F` - A vibrant green used for CTAs and highlights
- **Background**: `#050505` - Deep black for the main background
- **Surface**: Elevated surfaces with subtle borders
- **Typography**: Inter font family with various weights

## 📦 Component Categories

### Dashboard Components
Components used in the main dashboard interface:
- **StatCard** - Display key metrics with trends
- **ProjectCard** - Show project status and clips
- **AIInsightCard** - AI-powered insights and recommendations

### UI Components
Core reusable UI elements:
- **StatusBadge** - Show status states (completed, pending, failed)
- **ProgressBar** - Visual progress indicators
- **Skeleton** - Loading placeholders
- **Switch** - Toggle switches for settings

### Project Components
Components for managing clips and projects:
- **ClipCard** - Display individual clips with actions
- **NFTCard** - Show NFT status and details

### Platform Components
Social media platform integration:
- **PlatformCard** - Connect and manage social platforms

### Wallet Components
Blockchain wallet integration:
- **WalletConnectButton** - Connect MetaMask and other wallets

## 🚀 Getting Started

### Installation

```bash
npm install
```

### Running Storybook

```bash
npm run storybook
```

### Building Storybook

```bash
npm run build-storybook
```

## 📖 Usage

Each component story includes:

- **Interactive controls** - Modify props in real-time
- **Multiple variants** - See all component states
- **Accessibility checks** - Built-in a11y testing
- **Documentation** - Auto-generated from TypeScript types

## 🎯 Best Practices

1. **Use semantic HTML** - All components follow accessibility guidelines
2. **Responsive design** - Components adapt to different screen sizes
3. **Dark mode first** - Designed for dark backgrounds
4. **Performance** - Optimized with React.memo where appropriate

## 🤝 Contributing

When adding new components:

1. Create the component in the appropriate directory
2. Add a `.stories.tsx` file with multiple variants
3. Include accessibility labels and ARIA attributes
4. Test with keyboard navigation
5. Document all props with TypeScript types
